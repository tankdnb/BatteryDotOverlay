# Backlog Improvements

Текущая версия backlog оформлена под релиз `0.0.8`.

## Исследование

Во время ревизии проекта были выделены несколько зон, которые сильнее всего влияют на удобство тестирования и сопровождения:

- запуск нескольких копий приложения приводил к неочевидному `KeyInUse` и создавал впечатление падения;
- у приложения не было постоянного лог-файла, поэтому после проблем на машине пользователя диагностика опиралась только на консоль;
- конфиг мог содержать некорректные значения, но приложение не давало отдельной команды для безопасной проверки настроек;
- release-пакетам не хватало явного описания улучшений и сервисных команд для диагностики.

Из этого исследования был собран рабочий backlog на текущую итерацию.

## Выполненный backlog

- [x] `BL-01` Безопасный старт и защита от второй копии приложения.
  Что сделано: добавлена блокировка повторного запуска по `overlay.overlay_key`, улучшены сообщения о конфликте и уточнена причина проблем со стартом.

- [x] `BL-02` Постоянное логирование в файл с ротацией.
  Что сделано: приложение теперь пишет отдельный лог-файл на каждый запуск, ограничивает число хранимых логов и перехватывает необработанные исключения.

- [x] `BL-03` Валидация и нормализация конфигурации.
  Что сделано: добавлена отдельная команда `--validate-config`, проверка диапазонов, резервные значения для пустых параметров и понятные предупреждения.

- [x] `BL-04` Доработка документации и release-пакета.
  Что сделано: обновлены `README.md`, инструкции на русском и английском, release notes и описание сервисных команд.

## Что именно улучшено в 0.0.8

- безопасный запуск без ложного ощущения «приложение сразу упало»;
- понятный путь к активному лог-файлу через `--show-log-path`;
- проверка конфига без запуска overlay через `--validate-config`;
- более надёжная диагностика проблем на стороне тестировщика;
- самодостаточный release-пакет с расширенным описанием изменений.

## Идеи на следующие итерации

Ниже идеи, которые не включались в текущий backlog, чтобы сохранить компактность релиза:

- добавить отдельный диагностический режим для отображения текущего источника батареи и зарядки прямо в overlay;
- вынести логику источников батареи в независимые провайдеры с более подробной телеметрией;
- добавить автоэкспорт диагностического отчёта рядом с логом для удобной отправки тестировщиком;
- сделать отдельный режим «только индикатор без консольного окна» для постоянного фонового использования.
