Battery Dot Overlay
Configuration reference

Main config file:
config\indicator.settings.json

Main sections

- overlay
  Overlay identity, size, distance, and offsets.

- visual
  Dot color, opacity, texture size, and circle shape parameters.

- behavior
  Blink threshold, blink frequency, polling rate, and charging-aware logic.

- battery
  Battery and charging data sources plus diagnostics.

- runtime
  Internal loop timing, file logging, and single-instance protection.

Most important parameters

- behavior.blink_below_percent
  Battery threshold below which the dot may appear.

- behavior.blink_only_when_not_charging
  When true, the dot blinks only if battery is below the threshold and the headset is not charging.

- battery.enable_adb_fallback
  Enables Android/ADB fallback via `adb shell dumpsys battery`.

- battery.adb_device_serial
  Optional ADB device serial when multiple Android devices are connected.

- runtime.enable_file_logging
  Enables persistent file logging.

- runtime.logs_directory
  Directory for log files.

- runtime.max_log_files
  Number of recent logs to keep.

- runtime.enforce_single_instance
  Blocks a second copy from starting with the same overlay key.

Useful commands

- `BatteryDotOverlay.exe --validate-config`
- `BatteryDotOverlay.exe --show-log-path`
- `BatteryDotOverlay.exe --diagnose-battery`
