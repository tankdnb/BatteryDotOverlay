# Battery Dot Overlay

Минималистичный `SteamVR/OpenVR` overlay для VR-шлема, который показывает маленькую полупрозрачную жёлтую точку в правом верхнем углу поля зрения, когда заряд встроенного аккумулятора шлема ниже заданного порога.

Текущая версия: `0.0.4`

## Что делает программа

Приложение создаёт head-locked overlay, то есть индикатор закреплён относительно взгляда пользователя и всегда остаётся в одном и том же месте изображения внутри шлема.

Логика работы простая:

- если заряд батареи шлема ниже `behavior.blink_below_percent`, индикатор показывается;
- если включено мигание, точка мигает с частотой `behavior.blink_frequency_hz`;
- если заряд равен порогу или выше, индикатор скрывается;
- все визуальные и поведенческие параметры настраиваются через файл `config/indicator.settings.json`;
- файл конфигурации поддерживает комментарии `//`, поэтому его можно использовать как встроенную документацию.

## Для чего это нужно

Проект сделан для максимально простого и ненавязчивого уведомления о заряде батареи прямо в VR без лишних окон, панелей, текста и сложного интерфейса.

Сценарий использования:

- пользователь хочет видеть только факт, что батарея начала снижаться ниже нужного уровня;
- предупреждение должно быть заметным, но минималистичным;
- индикатор должен быть всегда в одном и том же месте и не зависеть от конкретного приложения в шлеме;
- настройки должны лежать в обычном конфиге и редактироваться без перекомпиляции.

## Основные возможности

- маленький круглый overlay-индикатор;
- полупрозрачный цвет настраивается из конфига;
- положение точки по горизонтали и вертикали настраивается из конфига;
- размер точки и расстояние от глаз настраиваются из конфига;
- частота мигания настраивается из конфига;
- порог появления индикатора по уровню заряда настраивается из конфига;
- поддерживаются комментарии в конфиге;
- есть self-contained release для передачи на тестирование без установки `.NET`;
- есть подробные инструкции на английском и русском языке;
- есть диагностика источника батареи через `OpenVR` и fallback для `PICO Connect`.

## Как это работает

Приложение использует `OpenVR companion overlay` и создаёт отдельный overlay-слой, который:

- инициализируется как `VRApplication_Overlay`;
- привязывается к `HMD` через `SetOverlayTransformTrackedDeviceRelative`;
- использует PNG-текстуру с круглой точкой;
- периодически опрашивает источник данных о батарее;
- показывает или скрывает overlay в зависимости от текущего процента заряда.

Источник батареи выбирается так:

1. Сначала приложение пытается получить заряд HMD через стандартные свойства `OpenVR`.
2. Если драйвер runtime не отдаёт батарею шлема напрямую, приложение может использовать fallback.
3. Для `PICO` fallback читает свежие события батареи из локальных логов `PICO Connect`.

Это сделано потому, что на некоторых runtime батарея шлема физически известна системе, но не публикуется в стандартное свойство `Prop_DeviceBatteryPercentage_Float`.

## Поддерживаемый сценарий

Основной целевой сценарий:

- Windows PC;
- установлен и запущен `SteamVR/OpenVR`;
- подключён VR-шлем;
- для `PICO` желательно, чтобы `PICO Connect` был запущен в фоне.

## Требования

Для запуска из готового релиза:

- Windows;
- `SteamVR` или совместимый `OpenVR` runtime;
- подключённый и активный VR-шлем.

Для запуска из исходников:

- `.NET 8 SDK`;
- Windows;
- `SteamVR/OpenVR`;
- исходники проекта.

## Ограничения

- приложение не работает само по себе без `SteamVR/OpenVR`, потому что это именно overlay поверх runtime;
- если VR-драйвер не отдаёт батарею HMD и нет поддерживаемого fallback, приложение не сможет показать корректный статус;
- на `PICO` для fallback требуется доступ к свежим логам `PICO Connect`;
- если одновременно запускать несколько копий программы с одинаковым `overlay_key`, `OpenVR` может вернуть ошибку `KeyInUse`;
- приложение не рисует текст, проценты или сложный UI, только минимальный визуальный сигнал.

## Конфигурация

Главный файл настроек:

```text
config/indicator.settings.json
```

Внутри файла уже есть подробные комментарии на русском языке. Основные разделы:

- `overlay` — положение, расстояние, размер и ключ overlay;
- `visual` — цвет, прозрачность и параметры формы круга;
- `behavior` — порог срабатывания, частота мигания и интервалы обновления;
- `battery` — параметры диагностики и fallback для `PICO Connect`;
- `runtime` — внутренняя частота цикла приложения.

Самые важные параметры:

- `behavior.blink_below_percent` — ниже какого процента показывать индикатор;
- `behavior.blink_frequency_hz` — частота мигания;
- `visual.color_hex` — цвет точки;
- `visual.opacity` — прозрачность точки;
- `overlay.horizontal_offset_meters` — смещение вправо или влево;
- `overlay.vertical_offset_meters` — смещение вверх или вниз;
- `overlay.width_in_meters` — размер точки;
- `overlay.distance_meters` — расстояние от глаз до индикатора.

Пример:

- `100.0` в `behavior.blink_below_percent` означает показывать индикатор при любом заряде ниже 100%;
- `90.0` означает показывать индикатор только ниже 90%;
- `0.0` в `behavior.blink_frequency_hz` означает не мигать, а просто держать точку видимой.

## Запуск из исходников

Запуск из папки проекта:

```powershell
dotnet run --project .\BatteryDotOverlay
```

Запуск с явным конфигом:

```powershell
dotnet run --project .\BatteryDotOverlay -- --settings .\BatteryDotOverlay\config\indicator.settings.json
```

Короткий тестовый запуск:

```powershell
dotnet run --project .\BatteryDotOverlay -- --duration-seconds 5
```

Запуск только для диагностики источника батареи:

```powershell
dotnet run --project .\BatteryDotOverlay -- --diagnose-battery
```

## Аргументы командной строки

Поддерживаемые аргументы:

- `--settings <path>` — использовать конкретный файл конфигурации;
- `--duration-seconds <n>` — автоматически завершить приложение через `n` секунд;
- `--diagnose-battery` — только вывести диагностику батареи и завершиться.

## Готовые релизы для тестирования

Каждая тестовая сборка выпускается как отдельный self-contained release.

Что входит в релиз:

- `BatteryDotOverlay.exe`;
- все нужные `.dll` для запуска без отдельной установки `.NET`;
- `openvr_api.dll`;
- `config\indicator.settings.json`;
- `START-HERE.txt`;
- `START-HERE-RU.txt`;
- `CONFIGURE-OVERLAY.txt`;
- `CONFIGURE-OVERLAY-RU.txt`;
- `RELEASE-INFO.txt`.

Что делает release-процесс:

- публикует автономную `win-x64` сборку;
- создаёт отдельную папку релиза по версии;
- собирает ZIP-архив для передачи тестировщику;
- добавляет инструкции по запуску и настройке.

Команды:

Собрать релиз текущей версии:

```powershell
.\Build-Release.cmd
```

Собрать следующую сборку с увеличением третьей цифры:

```powershell
.\Build-Release.cmd -BumpBuild
```

Сменить `major/minor` и автоматически сбросить номер сборки на `1`:

```powershell
.\Build-Release.cmd -Major 0 -Minor 1
```

Пересобрать уже существующий релиз той же версии:

```powershell
.\Build-Release.cmd -OverwriteExisting
```

Результат:

```text
BatteryDotOverlay\releases\BatteryDotOverlay-<version>-win-x64\
BatteryDotOverlay\releases\BatteryDotOverlay-<version>-win-x64.zip
```

## Диагностика и отладка

Приложение пишет в консоль:

- путь к используемому конфигу;
- номер версии;
- список устройств OpenVR при диагностике;
- источник батареи;
- текущий статус индикатора.

Типовые сообщения:

- `Battery value unavailable` — батарея не была получена ни из OpenVR, ни из fallback;
- `PICO Connect headset battery: ...` — заряд успешно получен из логов `PICO Connect`;
- `CreateOverlay failed: KeyInUse` — уже существует overlay с тем же `overlay_key`;
- `OpenVR init failed` — runtime недоступен или неправильно инициализирован.

Если индикатор не виден:

1. Убедитесь, что `SteamVR` запущен.
2. Убедитесь, что шлем подключён и активен.
3. Для `PICO` проверьте, что работает `PICO Connect`.
4. Запустите диагностику через `--diagnose-battery`.
5. Проверьте, не слишком ли маленький размер точки и не слишком ли далеко она сдвинута.
6. Проверьте, что заряд действительно ниже `behavior.blink_below_percent`.

## Структура проекта

Основные файлы:

- `Program.cs` — цикл приложения, запуск overlay и управление опросом батареи;
- `BatteryIndicatorOverlay.cs` — создание и управление OpenVR overlay;
- `BatteryTelemetry.cs` — чтение и выбор источника батареи;
- `PicoConnectBatteryLogReader.cs` — fallback для чтения батареи из логов `PICO Connect`;
- `BatteryDotSettings.cs` — модель конфигурации, загрузка и нормализация настроек;
- `DotTextureFactory.cs` — генерация текстуры круглой точки;
- `config/indicator.settings.json` — основной конфиг;
- `scripts/Build-Release.ps1` — сборка self-contained релизов;
- `Version.props` — источник истины для версии приложения.

## Версионирование

Схема версий:

```text

```

Правило:

- первая сборка: `0.0.1`;
- каждая следующая сборка увеличивает только третью цифру;
- при смене `major` или `minor` номер сборки начинается заново с `1`.

Источник истины для версии:

```text

```

Полезные команды:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Bump-BuildVersion.ps1
```

```powershell

```

## Основа реализации

Проект опирается на собранную информацию и кодовую базу `VR-app`:

- переиспользован `OpenVrRuntime`;
- использован подход companion overlay для `OpenVR`;
- для текстуры индикатора используется file-backed PNG overlay.

Это позволило сделать отдельную небольшую утилиту без лишней логики, но с совместимым и устойчивым способом интеграции в `OpenVR`.
