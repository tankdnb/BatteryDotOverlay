# Battery Dot Overlay

Минималистичный `SteamVR/OpenVR` overlay для VR-шлема. Приложение показывает маленькую полупрозрачную жёлтую точку в правом верхнем углу поля зрения, когда заряд встроенного аккумулятора шлема ниже заданного порога.

Текущая версия: `0.0.8`

## Что улучшено в 0.0.8

- добавлена защита от повторного запуска второй копии приложения с тем же `overlay_key`;
- добавлено постоянное логирование в файл с ротацией;
- появились сервисные команды `--validate-config`, `--show-log-path` и `--help`;
- добавлена валидация и нормализация настроек с понятными предупреждениями;
- улучшены сообщения о причинах неуспешного старта, особенно для `KeyInUse`;
- оформлен backlog улучшений в [BACKLOG.md](./BACKLOG.md).

## Что делает программа

Приложение создаёт head-locked overlay, то есть индикатор закреплён относительно взгляда пользователя и остаётся в одном и том же месте изображения внутри шлема.

Основная логика:

- если заряд батареи шлема ниже `behavior.blink_below_percent`, индикатор может включиться;
- по умолчанию индикатор мигает только тогда, когда шлем не заряжается;
- если определить статус зарядки не удалось, приложение сохраняет старое поведение и ориентируется только на процент батареи;
- если заряд равен порогу или выше, индикатор скрывается;
- все параметры вынесены в `config/indicator.settings.json`;
- конфиг поддерживает комментарии `//`.

## Основные возможности

- маленький круглый overlay-индикатор без текста и лишнего UI;
- настройка цвета, прозрачности, размера и положения;
- настройка порога заряда для предупреждения;
- настройка частоты мигания;
- учёт состояния зарядки шлема с возможностью отключить этот режим через конфиг;
- приоритетный Android/ADB fallback для батареи и питания, если runtime не отдаёт HMD-данные напрямую;
- запасной fallback для `PICO Connect`, если более прямые источники недоступны;
- self-contained release для передачи на тестирование без установки `.NET`;
- инструкции на русском и английском языке.

## Источники данных о батарее и зарядке

Приложение использует несколько источников в порядке приоритета:

1. `OpenVR`
   Сначала приложение пытается читать `Prop_DeviceBatteryPercentage_Float` и `Prop_DeviceIsCharging_Bool`.

2. `ADB battery service`
   Если `OpenVR` не помогает, приложение может использовать `adb shell dumpsys battery`.
   Это основной универсальный fallback для Android-шлемов.

3. `PICO Connect`
   Если `OpenVR` и `ADB` не дали результата, приложение может использовать локальные логи `PICO Connect`.
   Для `PICO Connect SDK` power-флаг дополнительно сверяется с трендом батареи, чтобы ложный `charging` не скрывал индикатор.

## Безопасный запуск

Начиная с `0.0.8` приложение защищается от повторного запуска:

- если уже работает другая копия с тем же `overlay_key`, новая копия не будет пытаться захватить overlay повторно;
- вместо неочевидного поведения приложение пишет понятную ошибку и показывает уже запущенные процессы;
- этот режим управляется параметром `runtime.enforce_single_instance`.

## Логи

Теперь приложение пишет лог-файл.

По умолчанию:

- логи создаются в папке `logs` рядом с `exe`;
- хранится ограниченное число последних файлов;
- путь и количество файлов задаются через `runtime.logs_directory` и `runtime.max_log_files`.

Полезная команда:

```powershell
BatteryDotOverlay.exe --show-log-path
```

## Конфигурация

Главный файл настроек:

```text
config/indicator.settings.json
```

Основные разделы:

- `overlay` — положение, размер, расстояние и ключ overlay;
- `visual` — цвет, прозрачность и параметры формы круга;
- `behavior` — порог срабатывания, частота мигания и логика учёта зарядки;
- `battery` — диагностика и fallback-источники (`OpenVR`, `ADB`, `PICO Connect`);
- `runtime` — частота цикла, логирование и защита от повторного запуска.

Самые важные параметры:

- `behavior.blink_below_percent` — ниже какого процента можно включать индикатор;
- `behavior.blink_only_when_not_charging` — учитывать ли зарядку шлема;
- `behavior.blink_frequency_hz` — частота мигания;
- `battery.enable_adb_fallback` — включить ли ADB fallback;
- `battery.adb_path` — путь к `adb.exe`, если нужно указать вручную;
- `battery.adb_device_serial` — serial Android-устройства, если их подключено несколько;
- `runtime.enable_file_logging` — писать ли лог-файл;
- `runtime.logs_directory` — папка для логов;
- `runtime.max_log_files` — сколько логов хранить;
- `runtime.enforce_single_instance` — защита от повторного запуска второй копии.

## Проверка конфига

Добавлена отдельная проверка настроек без запуска overlay:

```powershell
BatteryDotOverlay.exe --validate-config
```

Команда:

- проверяет значения в конфиге;
- печатает предупреждения, если параметры были скорректированы;
- показывает итоговую папку логов и найденный `adb.exe`.

## Запуск из исходников

Из папки проекта:

```powershell
dotnet run --project .\BatteryDotOverlay
```

С явным конфигом:

```powershell
dotnet run --project .\BatteryDotOverlay -- --settings .\BatteryDotOverlay\config\indicator.settings.json
```

Короткий тестовый запуск:

```powershell
dotnet run --project .\BatteryDotOverlay -- --duration-seconds 5
```

Диагностика батареи и зарядки:

```powershell
dotnet run --project .\BatteryDotOverlay -- --diagnose-battery
```

Проверка конфига:

```powershell
dotnet run --project .\BatteryDotOverlay -- --validate-config
```

## Аргументы командной строки

- `--settings <path>` — использовать конкретный файл конфигурации;
- `--duration-seconds <n>` — автоматически завершить приложение через `n` секунд;
- `--diagnose-battery` — вывести диагностику батареи и зарядки и завершиться;
- `--validate-config` — проверить конфиг и вывести сервисную информацию;
- `--show-log-path` — показать активный лог-файл;
- `--help` — показать справку.

## Требования

Для запуска готового релиза:

- Windows;
- установленный и запущенный `SteamVR` или совместимый `OpenVR` runtime;
- подключённый и активный VR-шлем.

Для лучшего определения статуса зарядки на Android-шлемах:

- включённая USB-отладка на шлеме;
- доступный `adb`;
- при нескольких Android-устройствах может понадобиться `battery.adb_device_serial`.

Для запуска из исходников:

- `.NET 8 SDK`;
- Windows;
- `SteamVR/OpenVR`;
- исходники проекта.

## Структура проекта

Основные файлы:

- `Program.cs` — цикл приложения, запуск overlay и сервисные режимы;
- `BatteryIndicatorOverlay.cs` — создание и управление `OpenVR` overlay;
- `BatteryTelemetry.cs` — чтение и объединение батареи и статуса зарядки;
- `AdbBatteryServiceReader.cs` — Android/ADB fallback для батареи и питания;
- `PicoConnectBatteryLogReader.cs` — запасной fallback для логов `PICO Connect`;
- `BatteryDotSettings.cs` — модель конфигурации и загрузка настроек;
- `BatteryDotSettingsValidator.cs` — валидация и нормализация настроек;
- `AppLogger.cs` — логирование в консоль и файл;
- `AppInstanceGuard.cs` — защита от повторного запуска;
- `config/indicator.settings.json` — основной конфиг;
- `scripts/Build-Release.ps1` — сборка self-contained релизов;
- `Version.props` — источник истины для версии приложения.

## Версионирование

Схема версий:

```text
major.minor.build
```

Правило:

- первая сборка: `0.0.1`;
- каждая следующая сборка увеличивает только третью цифру;
- при смене `major` или `minor` номер сборки начинается заново с `1`.

Источник истины для версии:

```text
Version.props
```

## Основа реализации

Проект опирается на собранную информацию и кодовую базу `VR-app`:

- переиспользован `OpenVrRuntime`;
- использован подход companion overlay для `OpenVR`;
- для текстуры индикатора используется file-backed PNG overlay;
- в материалах `VR-app` использовались идеи device-monitor overlay и диагностики батареи/зарядки.
